========================================================================
2. MODULE MAP
========================================================================

    [Scheduler / Entrypoint]
    main.py
      - Small launcher.
      - Supports one-shot execution via run_once().
      - Contains an optional run_loop() scheduler for repeated runs.

    [Pipeline Orchestrator]
    content_machine.py
      - Central coordinator for the full content pipeline.
      - Generates prompts, images, audio clips, video, and post upload.
      - Reads MUSIC_DIRECTORY from environment.
      - Owns output naming and directory creation for images/, audio/, videos/.

    [Prompt / LLM Layer]
    prompt_gen.py
      - Defines the Greek-mythology meme prompt strategy.
      - Builds a 4-step Ollama conversation:
          1) Theme
          2) Visual Concept
          3) Character Context
          4) Caption
      - Converts those outputs into the final image prompt.
      - create_prompt() is a standalone helper path used outside the main pipeline.

    [Image Generation Layer]
    image_generator.py
      - Calls OpenAI image generation with model "gpt-image-1".
      - Saves generated image bytes to disk.
      - Includes image reformat helper for 1080x1920 output.
      - Also exposes a standalone generate_image() path.

    [Audio Analysis Layer]
    updateAudio.py
      - Loads full audio.
      - Detects beats and bar alignment.
      - Computes repetition, RMS, and spectral centroid.
      - Scores candidate clip windows.
      - Exports the selected 20s clip.

    [Music Selection Utility]
    utils/findmusic.py
      - Picks a random .wav file from MUSIC_DIRECTORY / audio/.

    [Video Assembly Layer]
    makeVideo.py
      - Uses moviepy to create a simple video from one image plus one audio file.

    [Publishing Layer]
    insta_post.py
      - Creates temporary local file hosting with Python HTTP server.
      - Exposes hosted media through ngrok.
      - Creates and publishes Instagram media containers.
      - Supports both image and video post flows.
      - Main pipeline currently uses the video/reels path.

    [Filename Utility]
    utils/santize.py
      - Sanitizes caption text for filesystem-safe filenames.

    [Auxiliary / Not In Main Flow]
    ollama_client.py
      - Lightweight direct HTTP client for Ollama.
      - Not used by main.py/content_machine.py right now.
      - Exists as an alternate integration path to the Ollama Python package.

========================================================================
4. EXTERNAL DEPENDENCIES
========================================================================

    Ollama
      - Used by prompt_gen.py through the ollama Python package.
      - Responsible for theme / concept / caption generation.

    OpenAI Images API
      - Used by image_generator.py.
      - Responsible for final image generation.

    Librosa + NumPy + SoundFile
      - Used by updateAudio.py.
      - Responsible for audio analysis and clip export.

    MoviePy
      - Used by makeVideo.py.
      - Responsible for video composition.

    ngrok
      - Used by insta_post.py.
      - Makes local media publicly reachable for Instagram ingestion.

    Instagram Graph API
      - Used by insta_post.py.
      - Responsible for media container creation, polling, and publish.

========================================================================
6. ARCHITECTURAL CHARACTER
========================================================================

    Style
      - Single-process, script-oriented pipeline.
      - Orchestration is centralized in one module.
      - Modules are function-based rather than class-based.

    Strengths
      - Clear linear workflow.
      - Easy to run locally end-to-end.
      - Media artifacts are stored explicitly on disk between steps.

    Constraints
      - Tight coupling between generation, rendering, and publishing.
      - Heavy dependence on live external services.
      - Limited separation between "generate asset" and "publish asset" concerns.
      - Minimal state management beyond files and environment variables.
